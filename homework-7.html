Homework 7: Random Walk & Binomial Convergence in Cybersecurity

Simulate server security trajectories as random walks, analyze their convergence to the binomial distribution, and explore real-world cybersecurity implications of probabilistic threat modeling.

Part A ‚Äî Theoretical Framework & Cybersecurity Context

1 Problem Statement

Security Scenario: A server receives weekly security updates for \(n\) weeks. There are \(m\) attackers (or \(m\) independent threat simulations) attempting to breach the system, each with independent probability \(p\) of success per week.

Scoring System:

  • +1 if the server remains secure in a week (no successful breach)
  • -1 if the server is breached in a week (attack succeeds)

The cumulative score after \(n\) weeks represents the net security posture: higher scores indicate fewer breaches, lower scores indicate more breaches.

üîí Cybersecurity Interpretation
What does this model represent? This simulates a realistic cybersecurity scenario where:
  • Each week represents a critical period (monitoring cycle, update window)
  • \(m\) represents either multiple independent attackers OR multiple simulation runs for risk assessment
  • \(p\) is the organizational vulnerability index: how likely the system can be breached despite defenses
Real-World Example: A financial institution runs weekly security audits on their web servers. With \(n=52\) weeks (1 year), \(m=1000\) independent threat actors (crawlers, automated scanners), and \(p=0.1\) (10% chance each actor breaches in a given week), the cumulative score models the institution's long-term security posture.
2 Random Walk Interpretation

The cumulative score process is a discrete-time random walk with specific security implications:

Random Walk Definition:
Let \(S_k\) denote the cumulative score after week \(k\): $$S_k = \sum_{i=1}^{k} X_i$$ Where \(X_i\) is the outcome of week \(i\): $$X_i = \begin{cases} +1 & \text{with probability } (1-p) \text{ (week is secure)}\\ -1 & \text{with probability } p \text{ (breach occurs)} \end{cases}$$ Starting condition: \(S_0 = 0\) (neutral starting position)
üîí What Each Component Means in Cybersecurity
\(S_k\) = Cumulative Security Score:
  • \(S_k > 0\): System has remained secure more weeks than breached
  • \(S_k = 0\): Equal breaches and secure weeks (critical vulnerability)
  • \(S_k < 0\): More breaches than secure weeks (system compromised)
\(p\) = Breach Probability: Represents organizational risk
  • p = 0.05: Strong defenses (95% weekly security rate)
  • p = 0.30: Moderate risk (30% weekly breach rate)
  • p = 0.50: Critical (equally likely to be breached or secure each week)
  • p = 0.70: Severe (70% weekly breach rate - likely compromised)
\(1-p\) = Security Success Rate: Probability the system survives the week unbreached
  • Higher \((1-p)\) means more robust security infrastructure
  • Lower \((1-p)\) indicates vulnerability to attacks
Random Walk Properties & Security Implications:
  • State Space: \(S_n \in \{-n, -n+2, \ldots, n-2, n\}\)
    ‚ûú Possible security outcomes range from total compromise (\(-n\)) to perfect security (\(+n\))
  • Step Size: ¬±1 (unit steps)
    ‚ûú Each week contributes equally to security posture (no compounding)
  • Bias: Biased if \(p \neq 0.5\); symmetric if \(p = 0.5\)
    ‚ûú Non-equal p/q creates systematic drift (upward if secure, downward if vulnerable)
  • Expected Position: \(\mathbb{E}[S_n] = n(1-2p)\)
    ‚ûú Long-term expected security score reveals organizational risk trajectory
  • Variance: \(\text{Var}(S_n) = 4np(1-p)\)
    ‚ûú Uncertainty in final outcome; higher variance = less predictable security
üîí Interpreting the Drift
If p < 0.5 (Upward Drift): System is generally SECURE
  • More weeks without breaches than with breaches
  • Expected score \(\mathbb{E}[S_n] > 0\) grows with time
  • Security posture improves; updates are effective
  • Risk management is working
If p = 0.5 (Symmetric Walk): System in EQUILIBRIUM
  • 50/50 chance of breach vs security each week
  • \(\mathbb{E}[S_n] = 0\) (no trend)
  • Unpredictable security; needs urgent intervention
  • System is critically vulnerable
If p > 0.5 (Downward Drift): System is VULNERABLE
  • More breaches than secure weeks
  • \(\mathbb{E}[S_n] < 0\) deteriorates over time
  • Security posture worsens; defenses overwhelmed
  • Critical risk: immediate remediation required
3 Connection to Binomial Distribution

The final score \(S_n\) is directly related to the binomial distribution, which models the number of breaches. This connection is fundamental to understanding threat distribution across multiple attackers.

Binomial Relationship & Security Meaning:
Let \(K\) = number of breaches (successful attacks) in \(n\) weeks: $$K \sim \text{Binomial}(n, p)$$ $$P(K = k) = \binom{n}{k} p^k (1-p)^{n-k}$$ Security Interpretation:
  • \(\binom{n}{k}\) = number of ways k breaches can be distributed across n weeks
  • \(p^k\) = probability of exactly k successful attacks
  • \((1-p)^{n-k}\) = probability of n-k weeks remaining secure
Score from Breach Count:
Secure weeks: \(n - K\)
Breached weeks: \(K\)

Final cumulative score: $$S_n = (\text{secure weeks}) - (\text{breached weeks}) = (n - K) - K = n - 2K$$ Solving for K (breaches): $$K = \frac{n - S_n}{2}$$ Probability of final score being exactly s: $$P(S_n = s) = \binom{n}{\frac{n-s}{2}} p^{(n-s)/2} (1-p)^{(n+s)/2}$$
üîí What the Binomial Distribution Models
Peak of Distribution: Most likely number of breaches
  • Located at \(K_{mode} \approx np\)
  • For n=50, p=0.3: expect ‚âà15 breaches in 50 weeks
  • Helps organizations plan incident response capacity
Tails of Distribution: Extreme scenarios
  • Left tail: Extremely lucky outcomes (very few breaches)
  • Right tail: Worst-case scenarios (many breaches)
  • Tail probabilities inform risk management and insurance
  • Example: P(K > 30) when expecting 15 = probability of disaster
Distribution Shape: Indicates organizational predictability
  • Narrow distribution: Predictable security behavior (good)
  • Wide distribution: Highly uncertain outcomes (risky)
  • Skewed distribution: Asymmetric vulnerability
Key Statistics from Binomial:
Expected number of breaches: $$\mathbb{E}[K] = np$$ Expected cumulative score: $$\mathbb{E}[S_n] = n - 2\mathbb{E}[K] = n - 2np = n(1 - 2p)$$ Variance of score: $$\text{Var}(S_n) = 4\text{Var}(K) = 4np(1-p)$$ Standard deviation (uncertainty): $$\sigma_{S_n} = 2\sqrt{np(1-p)}$$
üîí Statistical Security Metrics
Expected Score \(\mathbb{E}[S_n]\): Average security outcome
  • Tells leadership: "On average, what's our security posture?"
  • Positive \(\mathbb{E}[S_n]\): Good (system generally secure)
  • Negative \(\mathbb{E}[S_n]\): Bad (system generally vulnerable)
Standard Deviation \(\sigma_{S_n}\): Unpredictability of outcomes
  • High \(\sigma\): Outcomes unpredictable; risky planning
  • Low \(\sigma\): Outcomes stable; reliable estimates
  • Used for confidence intervals: ¬±3\(\sigma\) = 99% probability range
Coefficient of Variation: Relative risk
  • \(\text{CV} = \sigma / |\mathbb{E}[S_n]|\)
  • High CV: Expected outcome has high uncertainty
  • Used to compare risk across different systems
4 Convergence: Why Simulations Match Theory

As both the number of weeks \(n\) and attackers \(m\) grow large, the empirical (simulated) distribution of cumulative scores converges to the theoretical binomial distribution. This is due to fundamental laws of probability.

Law of Large Numbers (LLN):
As the number of simulated trajectories \(m \to \infty\), the empirical frequency of each score converges to its theoretical probability: $$\frac{\text{count}(S_n = s)}{m} \xrightarrow{m \to \infty} P(S_n = s)$$ In practical terms:
  • 100 attackers: Rough estimate of distribution
  • 1,000 attackers: Good estimate
  • 10,000+ attackers: Excellent match to theory
Central Limit Theorem (CLT) for Large n:
For large \(n\), the distribution of \(S_n\) approaches a normal (Gaussian) distribution: $$\frac{S_n - \mathbb{E}[S_n]}{\sigma_{S_n}} = \frac{S_n - n(1-2p)}{2\sqrt{np(1-p)}} \xrightarrow{d} \mathcal{N}(0, 1)$$ Therefore (approximation): $$S_n \approx \mathcal{N}(n(1-2p), 4np(1-p))$$
üîí Why Convergence Matters for Cybersecurity
Predictability: With enough historical data (large m and n), we can reliably forecast security outcomes
  • Set realistic SLAs based on binomial probabilities
  • Allocate incident response resources efficiently
  • Plan budget for security improvements
Risk Quantification: Use normal approximation for confidence intervals
  • 95% of outcomes fall within \(\mathbb{E}[S_n] \pm 2\sigma_{S_n}\)
  • Identify exceptionally good or bad scenarios
  • Example: Only 5% chance score drops below -20
Comparing Systems: Use theoretical parameters to compare different security stances
  • System A: p=0.1 ‚Üí \(\mathbb{E}[S_{52}]\) = 41.6 (excellent)
  • System B: p=0.3 ‚Üí \(\mathbb{E}[S_{52}]\) = 20.8 (good)
  • System C: p=0.5 ‚Üí \(\mathbb{E}[S_{52}]\) = 0 (critical)
